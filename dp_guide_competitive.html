Dynamic Programming là gì?

Tư duy cốt lõi trước khi bắt đầu bất kỳ bài DP nào

Định nghĩa

Quy hoạch động (Dynamic Programming)

DP là kỹ thuật thiết kế thuật toán giải quyết bài toán bằng cách chia nhỏ thành các bài toán con nhỏ hơn, lưu kết quả để tránh tính lại. Hai đặc trưng thiết yếu:
  • Optimal Substructure (Cấu trúc con tối ưu): Nghiệm tối ưu của bài toán lớn được xây dựng từ nghiệm tối ưu của các bài toán con.
  • Overlapping Subproblems (Bài toán con lặp lại): Các bài toán con xuất hiện nhiều lần, cho phép memoization.
DP không phải là đệ quy — DP là tư duy quy hoạch trạng thái. Có thể cài đặt bằng top-down (đệ quy + memo) hoặc bottom-up (bảng DP).

🧠 Tư duy 4 bước giải DP

1
Định nghĩa trạng thái
dp[i], dp[i][j], dp[i][j][k]... Hỏi: "dp[...] có nghĩa là gì?"
2
Tìm công thức chuyển
dp[i] = f(dp[i-1], dp[i-2], ...). Xét tất cả "quyết định cuối cùng" dẫn đến trạng thái này.
3
Xác định base case
dp[0] = ?, dp[1] = ?... Trường hợp nhỏ nhất có thể xác định trực tiếp.
4
Xác định thứ tự tính
Đảm bảo khi tính dp[i], tất cả giá trị cần thiết đã được tính trước.

Quy trình học DP hiệu quả

  • Giai đoạn 1: Học từng dạng, hiểu sâu 1 bài trước khi sang bài khác. Đừng "collect" bài mà không hiểu.
  • Giai đoạn 2: Luyện nhận dạng — nhìn đề bài → biết ngay dạng DP nào.
  • Giai đoạn 3: Luyện biến tướng — cùng 1 dạng nhưng thêm ràng buộc, đòi hỏi tối ưu thêm.
  • Giai đoạn 4: Combine — bài khó thường kết hợp 2-3 kỹ thuật DP hoặc DP với cấu trúc dữ liệu.

01 · Dãy con tăng dài nhất
Bài toán kinh điển #1

LIS — Longest Increasing Subsequence

Cho dãy a[1..n], tìm dãy con tăng nghiêm ngặt dài nhất.

Trạng thái: dp[i] = độ dài dãy con tăng dài nhất kết thúc tại vị trí i.
Chuyển: dp[i] = max(dp[j] + 1) với mọi j < i và a[j] < a[i].
Kết quả: max(dp[1..n]).
dp[i] = max{ dp[j] + 1 | j < i, a[j] < a[i] }, dp[i] ≥ 1
C++ · LIS O(n²)
#include <bits/stdc++.h> using namespace std; int main() { int n; cin >> n; vector<int> a(n), dp(n, 1); for (int i = 0; i < n; i++) cin >> a[i]; for (int i = 1; i < n; i++) for (int j = 0; j < i; j++) if (a[j] < a[i]) dp[i] = max(dp[i], dp[j] + 1); cout << *max_element(dp.begin(), dp.end()); return 0; } // ─── LIS O(n log n) — dùng binary search ─── int LIS_fast(vector<int>& a) { vector<int> tails; // tails[i] = phần tử nhỏ nhất để LIS độ dài i+1 for (int x : a) { auto it = lower_bound(tails.begin(), tails.end(), x); if (it == tails.end()) tails.push_back(x); else *it = x; } return tails.size(); }
Kinh nghiệm

Các biến thể LIS thường gặp ở cấp tỉnh

  • LIS không giảm (non-decreasing): Đổi a[j] < a[i] thành a[j] <= a[i]. Trong binary search đổi lower_bound → upper_bound.
  • Đếm số dãy LIS: Thêm mảng cnt[i] = số cách để đạt dp[i].
  • LIS theo 2 chiều: Sắp xếp theo 1 chiều, LIS theo chiều còn lại (bài bao phong bì, xếp hộp...).
  • LNDS (Longest Non-Decreasing Subsequence): Sắp xếp + LCS với dãy đã sort.
02 · Bài toán Balo
Bài toán kinh điển #2

0/1 Knapsack — Balo 0/1

Có n vật, vật i có trọng lượng w[i] và giá trị v[i]. Balo sức chứa W. Chọn các vật (mỗi vật dùng tối đa 1 lần) để tổng giá trị max, không vượt quá W.

Trạng thái: dp[j] = giá trị max với balo sức chứa j.
Chuyển: Duyệt j từ W xuống w[i]: dp[j] = max(dp[j], dp[j-w[i]] + v[i]).
C++ · 0/1 Knapsack (Space Optimized)
// 0/1 Knapsack — bộ nhớ O(W) int knapsack01(int n, int W, vector<int>& w, vector<int>& v) { vector<int> dp(W + 1, 0); for (int i = 0; i < n; i++) for (int j = W; j >= w[i]; j--) // duyệt NGƯỢC để mỗi vật dùng đúng 1 lần dp[j] = max(dp[j], dp[j - w[i]] + v[i]); return dp[W]; } // Unbounded Knapsack (mỗi vật dùng vô hạn lần) int knapsackUnbounded(int n, int W, vector<int>& w, vector<int>& v) { vector<int> dp(W + 1, 0); for (int i = 0; i < n; i++) for (int j = w[i]; j <= W; j++) // duyệt XUÔI để dùng nhiều lần dp[j] = max(dp[j], dp[j - w[i]] + v[i]); return dp[W]; } // Bounded Knapsack (vật i dùng tối đa cnt[i] lần) — binary grouping int knapsackBounded(int n, int W, vector<int>& w, vector<int>& v, vector<int>& cnt) { vector<int> dp(W + 1, 0); for (int i = 0; i < n; i++) { // Tách vật i thành các nhóm: 1, 2, 4, ..., rem int rem = cnt[i]; for (int k = 1; rem > 0; k *= 2) { int take = min(k, rem); rem -= take; int wi = take * w[i], vi = take * v[i]; for (int j = W; j >= wi; j--) dp[j] = max(dp[j], dp[j - wi] + vi); } } return dp[W]; }
⚠️ Lỗi hay gặp

Nhầm hướng duyệt trong Knapsack

  • 0/1 Knapsack: Duyệt j từ W xuống w[i] — bảo đảm mỗi vật chỉ được chọn 1 lần vì giá trị cũ chưa bị ghi đè.
  • Unbounded Knapsack: Duyệt j từ w[i] lên W — cho phép dùng vật nhiều lần vì dp[j-w[i]] đã được cập nhật trong vòng lặp hiện tại.
  • Đây là lỗi cực kỳ phổ biến khi mới học DP, và điểm mất thường toàn bộ.
03 · LCS và Edit Distance
🔗LCS — Chuỗi con chung dài nhất
dp[i][j] = LCS của a[1..i] và b[1..j].
Nếu a[i]==b[j]: dp[i][j] = dp[i-1][j-1]+1
Còn lại: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
O(nm)2D DPchuỗi
✏️Edit Distance — Khoảng cách chỉnh sửa
dp[i][j] = số phép (insert/delete/replace) nhỏ nhất biến a[1..i] thành b[1..j].
Nếu a[i]==b[j]: dp[i][j] = dp[i-1][j-1]
Còn lại: dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
O(nm)chuỗiLevenshtein
🏔️Tam giác số (Triangle DP)
dp[i][j] = giá trị max đường đi từ đỉnh đến ô (i,j).
dp[i][j] = a[i][j] + max(dp[i-1][j-1], dp[i-1][j]).
Bài toán grid DP 2D cơ bản nhất.
grid DPtop-down
💰Đổi tiền (Coin Change)
Cho n loại tiền coin[]. Đổi tổng S với ít đồng nhất.
dp[j] = min số đồng để tổng j.
dp[j] = min(dp[j-coin[i]]+1) với mọi coin[i] ≤ j.
unboundedO(nS)
C++ · LCS + Truy vết
#include <bits/stdc++.h> using namespace std; string LCS(string& a, string& b) { int n = a.size(), m = b.size(); vector<vector<int>> dp(n+1, vector<int>(m+1, 0)); for (int i = 1; i <= n; i++) for (int j = 1; j <= m; j++) { if (a[i-1] == b[j-1]) dp[i][j] = dp[i-1][j-1] + 1; else dp[i][j] = max(dp[i-1][j], dp[i][j-1]); } // Truy vết string lcs = ""; int i = n, j = m; while (i > 0 && j > 0) { if (a[i-1] == b[j-1]) { lcs += a[i-1]; i--; j--; } else if (dp[i-1][j] > dp[i][j-1]) i--; else j--; } reverse(lcs.begin(), lcs.end()); return lcs; }

01 · Bitmask DP
Kỹ thuật quan trọng

Bitmask DP — DP trên tập con

Dùng khi cần mô tả tập con của n phần tử (n ≤ 20). Mỗi bit trong mask biểu diễn việc phần tử tương ứng được chọn hay chưa.

Bài toán người bán hàng (TSP): dp[mask][i] = chi phí đường đi nhỏ nhất xuất phát từ 0, đi qua các thành phố trong mask, kết thúc tại i.

Thao tác bit cần nhớ:
  • mask | (1<<i) — thêm phần tử i vào tập
  • mask & ~(1<<i) — xóa phần tử i khỏi tập
  • (mask >> i) & 1 — kiểm tra bit i
  • mask & (mask-1) — xóa bit 1 thấp nhất
  • __builtin_popcount(mask) — đếm số bit 1
C++ · TSP với Bitmask DP
// TSP: n thành phố, tìm chu trình ngắn nhất qua mọi thành phố // dp[mask][i] = chi phí nhỏ nhất đến thành phố i, đã đi qua mask const int INF = 1e9; int TSP(int n, vector<vector<int>>& dist) { int FULL = (1 << n) - 1; vector<vector<int>> dp(1 << n, vector<int>(n, INF)); dp[1][0] = 0; // xuất phát từ thành phố 0 for (int mask = 1; mask <= FULL; mask++) for (int u = 0; u < n; u++) { if (dp[mask][u] == INF) continue; if (!((mask >> u) & 1)) continue; // u không trong mask for (int v = 0; v < n; v++) { if ((mask >> v) & 1) continue; // v đã đi rồi int nmask = mask | (1 << v); dp[nmask][v] = min(dp[nmask][v], dp[mask][u] + dist[u][v]); } } int ans = INF; for (int u = 1; u < n; u++) ans = min(ans, dp[FULL][u] + dist[u][0]); return ans; } // Duyệt mọi subset của mask — O(3^n) tổng thể for (int sub = mask; sub > 0; sub = (sub - 1) & mask) { // xử lý sub } // SOS DP (Sum over Subsets) — tính tổng/max/min trên mọi tập con // f[mask] = tổng giá trị tất cả subset của mask for (int i = 0; i < n; i++) for (int mask = 0; mask < (1<<n); mask++) if ((mask >> i) & 1) f[mask] += f[mask ^ (1<<i)];
02 · Interval DP
Kỹ thuật quan trọng

Interval DP — DP trên đoạn [l, r]

dp[l][r] biểu diễn kết quả tối ưu trên đoạn [l..r].
Duyệt độ dài đoạn từ nhỏ đến lớn. Với mỗi đoạn [l,r], thử mọi điểm chia k.

Công thức tổng quát: dp[l][r] = min/max{ f(dp[l][k], dp[k+1][r]) | l ≤ k < r }

Bài toán kinh điển:
  • Matrix Chain Multiplication — nhân chuỗi ma trận
  • Optimal BST — cây tìm kiếm nhị phân tối ưu
  • Burst Balloons (Nổ bong bóng)
  • Strange Printer
dp[l][r] = min{ dp[l][k] + dp[k+1][r] + cost(l, r) | l ≤ k < r }
C++ · Interval DP — Nhân chuỗi ma trận
// Matrix Chain Multiplication // n ma trận A1...An, ma trận Ai có kích thước dim[i-1] × dim[i] // Tìm thứ tự nhân để tổng phép nhân ít nhất int matrixChain(int n, vector<int>& dim) { // dp[l][r] = số phép nhân ít nhất để nhân A_l...A_r vector<vector<int>> dp(n, vector<int>(n, 0)); for (int len = 2; len <= n; len++) { // duyệt độ dài đoạn for (int l = 0; l + len - 1 < n; l++) { int r = l + len - 1; dp[l][r] = INT_MAX; for (int k = l; k < r; k++) { // điểm chia int cost = dp[l][k] + dp[k+1][r] + dim[l] * dim[k+1] * dim[r+1]; dp[l][r] = min(dp[l][r], cost); } } } return dp[0][n-1]; } // Template interval DP tổng quát for (int len = 2; len <= n; len++) for (int l = 1; l + len - 1 <= n; l++) { int r = l + len - 1; dp[l][r] = INF; for (int k = l; k < r; k++) dp[l][r] = min(dp[l][r], dp[l][k] + dp[k+1][r] + cost(l, r, k)); }
03 · Tree DP
Kỹ thuật quan trọng

Tree DP — Quy hoạch động trên cây

DP trên cây thường được thực hiện bằng DFS từ gốc.
Trạng thái: dp[v][state] = kết quả tối ưu cho cây con gốc v, với trạng thái state.
Chuyển: Gộp kết quả từ các cây con con.

Dạng phổ biến nhất: dp[v][0/1] = kết quả khi v không chọn / có chọn.
Bài toán độc lập tối đa (Maximum Independent Set) trên cây:
  • dp[v][1] = chọn v + tổng dp[u][0] với u ∈ children(v)
  • dp[v][0] = tổng max(dp[u][0], dp[u][1]) với u ∈ children(v)
C++ · Tree DP — Maximum Independent Set + Knapsack on Tree
// 1. Maximum Independent Set on Tree int dp[100005][2]; // dp[v][0/1] int val[100005]; vector<int> adj[100005]; void dfs(int v, int par) { dp[v][1] = val[v]; // chọn v dp[v][0] = 0; // không chọn v for (int u : adj[v]) { if (u == par) continue; dfs(u, v); dp[v][1] += dp[u][0]; // chọn v → con không được chọn dp[v][0] += max(dp[u][0], dp[u][1]); // không chọn v → con tự do } } // 2. Knapsack on Tree — chọn tối đa k đỉnh để tổng giá trị max // dp[v][j] = giá trị max khi chọn j đỉnh trong cây con gốc v void dfsKnap(int v, int par, int K, vector<int> dp[], vector<int> adj[], int* val) { dp[v].assign(K + 1, 0); dp[v][1] = val[v]; // chỉ chọn v int sz = 1; // kích thước cây con hiện tại for (int u : adj[v]) { if (u == par) continue; dfsKnap(u, v, K, dp, adj, val); // Merge dp[v] với dp[u] — knapsack 2 chiều for (int j = min(sz + (int)dp[u].size()-1, K); j >= 0; j--) for (int k = min(j-1, (int)dp[u].size()-1); k >= 0; k--) dp[v][j] = max(dp[v][j], dp[v][j-k-1] + dp[u][k]); sz += dp[u].size() - 1; } }
04 · Digit DP
Kỹ thuật quan trọng

Digit DP — DP theo chữ số

Đếm số nguyên trong [L, R] thỏa một tính chất nào đó. Sử dụng kỹ thuật f(R) - f(L-1).

Trạng thái điển hình: dp[pos][tight][state] với:
  • pos: đang xét chữ số thứ mấy
  • tight: có đang bị ràng buộc bởi giới hạn trên không
  • state: trạng thái thuộc tính cần theo dõi (tổng chữ số, chữ số đã dùng...)
C++ · Digit DP Template
// Đếm số nguyên trong [0, N] có tổng chữ số chia hết cho k int k; string num; int memo[20][2][100]; // [pos][tight][rem] int solve(int pos, bool tight, int rem) { if (pos == (int)num.size()) return rem == 0; if (memo[pos][tight][rem] != -1) return memo[pos][tight][rem]; int limit = tight ? (num[pos] - '0') : 9; int res = 0; for (int d = 0; d <= limit; d++) { bool newTight = tight && (d == limit); res += solve(pos + 1, newTight, (rem + d) % k); } return memo[pos][tight][rem] = res; } int count(int N) { num = to_string(N); memset(memo, -1, sizeof(memo)); return solve(0, true, 0); } // Đếm trong [L, R] int ans = count(R) - count(L - 1);

01 · DP trên DAG
Kỹ thuật

DP trên DAG — Đồ thị có hướng không có chu trình

Bất kỳ bài DP nào có thể được biểu diễn như tìm đường đi tối ưu trên DAG.
Ý tưởng chính: Topo sort → DP theo thứ tự topo.
dp[v] = kết quả tối ưu khi kết thúc/bắt đầu tại v.

Ứng dụng:
  • Đường đi dài nhất/ngắn nhất trên DAG
  • Đếm số đường đi từ s đến t
  • DP memoization trên đệ quy (state space là DAG ẩn)
C++ · Longest Path on DAG
// Đường đi dài nhất trên DAG — topo sort + DP int n, m; vector<int> adj[100005]; int indeg[100005], dp[100005], w[100005]; int longestPath() { queue<int> q; for (int i = 1; i <= n; i++) if (indeg[i] == 0) { q.push(i); dp[i] = w[i]; } int ans = 0; while (!q.empty()) { int v = q.front(); q.pop(); ans = max(ans, dp[v]); for (int u : adj[v]) { dp[u] = max(dp[u], dp[v] + w[u]); if (--indeg[u] == 0) q.push(u); } } return ans; } // DP memoization trên state space ẩn (DAG ngầm) map<tuple<int,int,int>, int> memo; int dp_solve(int i, int j, int k) { if (/* base case */ i == n) return 0; auto key = make_tuple(i, j, k); if (memo.count(key)) return memo[key]; // tính toán và lưu vào memo return memo[key] = max(dp_solve(i+1, j, k), /* ... */ 0); }
02 · Sliding Window / Deque DP
Tối ưu hóa DP

Monotone Deque Optimization

Dùng khi công thức chuyển có dạng:
dp[i] = min/max{ dp[j] | j ∈ [i-K, i-1] } + cost(i)

Dùng deque (double-ended queue) duy trì min/max trên cửa sổ trượt, tối ưu từ O(nK) → O(n).

Nguyên tắc deque:
  • Front: luôn là phần tử min/max trong cửa sổ hiện tại
  • Pop front khi chỉ số ra khỏi cửa sổ [i-K, i-1]
  • Pop back khi thêm phần tử mới: đảm bảo tính đơn điệu
C++ · Deque DP — dp[i] = min{dp[j] | j∈[i-K, i-1]} + a[i]
#include <deque> using namespace std; // dp[i] = min{ dp[j] | max(0, i-K) <= j < i } + a[i] vector<int> dequeDP(vector<int>& a, int K) { int n = a.size(); vector<int> dp(n, INT_MAX); deque<int> dq; // lưu chỉ số, front = chỉ số cho giá trị dp nhỏ nhất dp[0] = a[0]; dq.push_back(0); for (int i = 1; i < n; i++) { // Xóa front nếu ra ngoài cửa sổ while (!dq.empty() && dq.front() < i - K) dq.pop_front(); // dp[i] = dp[front] + a[i] if (!dq.empty()) dp[i] = dp[dq.front()] + a[i]; // Thêm i vào deque, duy trì tính đơn điệu tăng while (!dq.empty() && dp[dq.back()] >= dp[i]) dq.pop_back(); dq.push_back(i); } return dp; } // Ứng dụng: Bài toán kho hàng — vận chuyển hàng với chi phí tối ưu // Bài: Maximum sum of subarray of size between L and R int maxSubarrayLR(vector<int>& a, int L, int R) { int n = a.size(); vector<int> prefix(n + 1, 0); for (int i = 0; i < n; i++) prefix[i+1] = prefix[i] + a[i]; int ans = INT_MIN; deque<int> dq; for (int i = L; i <= n; i++) { while (!dq.empty() && dq.front() < i - R) dq.pop_front(); if (!dq.empty()) ans = max(ans, prefix[i] - prefix[dq.front()]); while (!dq.empty() && prefix[dq.back()] >= prefix[i-L]) dq.pop_back(); dq.push_back(i - L); } return ans; }
03 · Divide and Conquer DP
Tối ưu hóa DP

D&C DP Optimization — O(n² → n log n)

Áp dụng khi công thức có dạng:
dp[i][j] = min{ dp[i-1][k] + cost(k+1, j) | k < j }
và hàm cost thỏa mãn Tính đơn điệu của điểm tối ưu: opt(i, j) ≤ opt(i, j+1) (opt không giảm theo j).

Cách cài: Chia đôi phạm vi j, tìm opt cho mid, đệ quy hai nửa. Mỗi lớp chia chỉ mất O(n) để duyệt cost → O(n log n) mỗi lớp × O(log n) lớp.
C++ · D&C DP Optimization
// Template D&C DP // dp[t][j] = min{ dp[t-1][k] + cost(k, j) | 0 <= k <= j } // Yêu cầu: opt[j] <= opt[j+1] int n, K; long long dp[505][100005]; long long prefix[100005]; // prefix sum để tính cost nhanh // cost(l, r) = (r-l+1) * (prefix[r] - prefix[l-1]) hoặc hàm gì đó long long cost(int l, int r) { return prefix[r] - prefix[l]; // ví dụ đơn giản } void solve(int t, int lo, int hi, int optLo, int optHi) { if (lo > hi) return; int mid = (lo + hi) / 2; int opt = optLo; dp[t][mid] = LLONG_MAX; for (int k = optLo; k <= min(mid, optHi); k++) { long long val = dp[t-1][k] + cost(k, mid); if (val < dp[t][mid]) { dp[t][mid] = val; opt = k; } } solve(t, lo, mid - 1, optLo, opt); solve(t, mid + 1, hi, opt, optHi); } // Gọi cho mỗi lớp t for (int t = 1; t <= K; t++) solve(t, 1, n, 1, n);
04 · Convex Hull Trick (Cơ bản)
Tối ưu hóa DP quan trọng

Convex Hull Trick (CHT) — Cơ bản

Dùng khi công thức có dạng: dp[i] = min{ a[j] * b[i] + c[j] }
Tức là mỗi j định nghĩa một đường thẳng y = a[j]*x + c[j], và với mỗi i ta cần tìm đường thẳng cho giá trị y nhỏ nhất tại x = b[i].

Dùng convex hull của tập đường thẳng (lower envelope) để truy vấn O(1) hoặc O(log n).

Điều kiện dùng CHT offline đơn giản:
  • Slope a[j] đơn điệu (sắp xếp được)
  • Query b[i] đơn điệu
Khi không đơn điệu → dùng Li Chao Tree hoặc CHT với binary search.
C++ · CHT — Minimum Line (slope monotone, query monotone)
// CHT: dp[i] = min{ m[j]*x[i] + b[j] } // slope m[j] tăng dần, query x[i] tăng dần → dùng deque, O(n) tổng typedef long long ll; struct Line { ll m, b; ll eval(ll x) { return m * x + b; } }; bool bad(Line l1, Line l2, Line l3) { // Đường l2 không bao giờ là min → loại bỏ return (l3.b - l1.b) * (l1.m - l2.m) <= (l2.b - l1.b) * (l1.m - l3.m); } deque<Line> hull; int ptr = 0; void addLine(ll m, ll b) { // slope tăng dần Line l = {m, b}; while (hull.size() >= 2 && bad(hull[hull.size()-2], hull.back(), l)) hull.pop_back(); hull.push_back(l); } ll queryMin(ll x) { // query x tăng dần while (hull.size() >= 2 && hull[0].eval(x) >= hull[1].eval(x)) hull.pop_front(); return hull.front().eval(x); } // Ví dụ: Bài toán chia k phần — chi phí chia [l, r] thành 1 phần là (sum)^2 // dp[i][j] = min cost chia a[1..i] thành j phần // dp[i][j] = min{ dp[k][j-1] + (prefix[i] - prefix[k])^2 }

01 · Knuth Optimization
Tối ưu hóa DP — O(n³ → n²)

Knuth-Yao Speedup

Tối ưu Interval DP từ O(n³) xuống O(n²).
Điều kiện: cost(a,c) + cost(b,d) ≤ cost(a,d) + cost(b,c) (quad. concave / monge condition) VÀ opt[i][j-1] ≤ opt[i][j] ≤ opt[i+1][j].

Khi đó: giới hạn k từ [l, r-1] → [opt[l][r-1], opt[l+1][r]].
Tổng số thao tác trở thành O(n²).

Bài toán điển hình: Optimal BST, chia mảng thành k nhóm với chi phí tổng.
C++ · Knuth Optimization
// Knuth: dp[l][r] = min{ dp[l][k] + dp[k+1][r] + w(l, r) } // opt[l][r]: điểm chia tối ưu int dp[505][505], opt[505][505]; int w[505][505]; // cost void knuth(int n) { // Base case for (int i = 1; i <= n; i++) { dp[i][i] = 0; opt[i][i] = i; } for (int len = 2; len <= n; len++) { for (int l = 1; l + len - 1 <= n; l++) { int r = l + len - 1; dp[l][r] = INT_MAX; // Chỉ duyệt từ opt[l][r-1] đến opt[l+1][r] for (int k = opt[l][r-1]; k <= opt[l+1][r]; k++) { int val = dp[l][k] + dp[k+1][r] + w[l][r]; if (val < dp[l][r]) { dp[l][r] = val; opt[l][r] = k; } } } } } // Cách tính w[l][r] phổ biến — w(i,j) = prefix[j] - prefix[i-1] // Tức là w(i,j) = tổng a[i..j] — thỏa điều kiện Knuth for (int i = 1; i <= n; i++) for (int j = i; j <= n; j++) w[i][j] = prefix[j] - prefix[i-1];
02 · Profile DP (Broken Profile DP)
DP trên lưới

Broken Profile DP — Ô lát gạch

Dùng để đếm số cách lát gạch trên bảng n×m với gạch 1×2 (domino) hoặc 2×1.
Trạng thái: dp[col][mask] = số cách lát đến cột col, với mask mô tả các ô ở cột col đã được "lấp đầy" từ cột trước (thò sang).

Yếu tố kỹ thuật:
  • Duyệt từng ô theo hàng-cột, mỗi bước xét ô tiếp theo
  • Mask chỉ cần n bits (n ≤ 20)
  • Phức tạp: O(m · 2^n · 2^n) hoặc O(m · 3^n) tùy cách cài
Được dùng trong các bài lát gạch, đếm cấu hình hợp lệ trên lưới.
C++ · Profile DP — Domino Tiling n×m
// Đếm số cách lát n×m bằng domino 1×2 // dp[mask] = số cách khi profile cột hiện tại là mask typedef long long ll; ll dominoCount(int n, int m) { vector<ll> dp(1 << n, 0), ndp; dp[0] = 1; auto go = [&](auto& self, int row, int oldMask, int newMask, vector<ll>& dp, vector<ll>& ndp) { if (row == n) { ndp[newMask] += dp[oldMask]; return; } if ((oldMask >> row) & 1) { // ô này đã được lấp từ cột trước self(self, row + 1, oldMask, newMask, dp, ndp); } else { // ô này trống // đặt gạch ngang (lấp ô này + ô bên phải ở cột sau) self(self, row + 1, oldMask, newMask | (1 << row), dp, ndp); // đặt gạch dọc (lấp ô này + ô bên dưới) if (row + 1 < n && !((oldMask >> (row+1)) & 1)) self(self, row + 2, oldMask, newMask, dp, ndp); } }; for (int col = 0; col < m; col++) { ndp.assign(1 << n, 0); for (int mask = 0; mask < (1 << n); mask++) if (dp[mask]) go(go, 0, mask, 0, dp, ndp); dp = ndp; } return dp[0]; }
03 · Li Chao Tree
Cấu trúc dữ liệu + DP

Li Chao Tree — CHT không đơn điệu

Khi slope và query không đơn điệu, dùng Li Chao Tree thay CHT thông thường.
Li Chao Tree là segment tree trên tập x-values, mỗi nút lưu đường thẳng "thắng" tại mid.

Độ phức tạp: O(log n) mỗi thao tác thêm đường thẳng hoặc query.
Tổng: O(n log n) cho n đường thẳng + n query.
C++ · Li Chao Tree (Minimum Lines)
const int MAXV = 200005; struct Line { long long m, b; }; const long long INF = 1e18; struct LiChaoTree { struct Node { Line line = {0, INF}; int l = 0, r = 0; }; vector<Node> tree; int lo, hi, root; LiChaoTree(int lo, int hi) : lo(lo), hi(hi) { tree.push_back({}); // node 0 là null root = newNode(); } int newNode() { tree.push_back({}); return tree.size() - 1; } long long eval(Line& l, long long x) { return l.m * x + l.b; } void addLine(int node, int lo, int hi, Line line) { int mid = (lo + hi) / 2; bool leftBetter = eval(line, lo) < eval(tree[node].line, lo); bool midBetter = eval(line, mid) < eval(tree[node].line, mid); if (midBetter) swap(tree[node].line, line); if (lo == hi) return; if (leftBetter != midBetter) { if (!tree[node].l) tree[node].l = newNode(); addLine(tree[node].l, lo, mid, line); } else { if (!tree[node].r) tree[node].r = newNode(); addLine(tree[node].r, mid + 1, hi, line); } } long long query(int node, int lo, int hi, long long x) { if (!node) return INF; int mid = (lo + hi) / 2; long long res = eval(tree[node].line, x); if (x <= mid) return min(res, query(tree[node].l, lo, mid, x)); else return min(res, query(tree[node].r, mid+1, hi, x)); } void add(Line l) { addLine(root, lo, hi, l); } long long get(long long x) { return query(root, lo, hi, x); } };

01 · Aliens Trick (Lambda / WQS Binary Search)
Kỹ thuật APIO kinh điển

Aliens Trick — Loại bỏ ràng buộc số lượng

Bài toán: tìm giá trị tối ưu khi chọn đúng k phần/nhóm, và hàm tối ưu theo k là lõm (concave).

Ý tưởng: Thay vì fix k, thêm một "phí" lambda cho mỗi phần chọn thêm.
Binary search lambda để số phần tối ưu = k.

Điều kiện áp dụng:
  • Hàm g(k) = kết quả tối ưu khi chọn k phần là hàm lõm (concave)
  • Bài toán relaxed (không fix k) có thể giải hiệu quả
Độ phức tạp: O(f(n) × log(max_cost)), với f(n) là DP không có ràng buộc k.
C++ · Aliens Trick Template
// Bài toán: IOI 2016 Aliens // Chia n điểm thành k nhóm, chi phí mỗi nhóm là f(l, r) // Tìm tổng chi phí nhỏ nhất typedef long long ll; typedef pair<ll, ll> pll; // {cost, count} // dp với chi phí thêm lambda cho mỗi nhóm pll solve_with_lambda(ll lambda) { // dp[i] = {min cost, min number of groups} khi xét i phần tử đầu // Tùy bài, viết DP cụ thể // Ví dụ với CHT hoặc deque vector<pll> dp(n + 1, {0LL, 0LL}); // ... DP code ở đây với chi phí thêm lambda mỗi khi bắt đầu nhóm mới ... return dp[n]; } int main() { int n, k; cin >> n >> k; ll lo = 0, hi = 1e13; while (lo < hi) { ll mid = (lo + hi) / 2; auto [cost, cnt] = solve_with_lambda(mid); // Nếu số nhóm tối ưu > k: tăng lambda (tốn thêm tiền khi tạo nhóm) if (cnt > k) lo = mid + 1; else hi = mid; } auto [cost, cnt] = solve_with_lambda(lo); // Đáp án: cost - k * lo (bù phần thêm vào) cout << cost - (ll)k * lo << "\n"; } // Lưu ý quan trọng: // - Khi có nhiều lambda cho cùng cnt = k, lấy lo nhỏ nhất để đáp án đúng // - Với k không có lambda tương ứng chính xác → cần xử lý thêm
02 · Segment Tree DP
Kỹ thuật kết hợp

DP + Segment Tree — Tối ưu Query Range

Khi công thức chuyển dp[i] cần query tối ưu trên một đoạn [l, r] phức tạp, dùng Segment Tree để lưu và cập nhật DP.

Pattern phổ biến:
  • dp[i] = max{ dp[j] + val[j][i] | j < i, condition } → Seg Tree Range Max Query
  • Kết hợp lazy propagation khi cần update đoạn
  • Merge sort tree khi cần query 2D
C++ · DP + Segment Tree — Maximum dp[j] với j trong đoạn
// dp[i] = max{dp[j] | j ∈ [L(i), R(i)]} + a[i] // Với mỗi i, L(i) và R(i) có thể khác nhau struct SegTree { int n; vector<long long> tree; SegTree(int n) : n(n), tree(4*n, LLONG_MIN/2) {} void update(int node, int lo, int hi, int pos, long long val) { if (lo == hi) { tree[node] = max(tree[node], val); return; } int mid = (lo + hi) / 2; if (pos <= mid) update(2*node, lo, mid, pos, val); else update(2*node+1, mid+1, hi, pos, val); tree[node] = max(tree[2*node], tree[2*node+1]); } long long query(int node, int lo, int hi, int l, int r) { if (r < lo || hi < l) return LLONG_MIN/2; if (l <= lo && hi <= r) return tree[node]; int mid = (lo + hi) / 2; return max(query(2*node, lo, mid, l, r), query(2*node+1, mid+1, hi, l, r)); } void upd(int pos, long long val) { update(1, 0, n-1, pos, val); } long long qry(int l, int r) { return query(1, 0, n-1, l, r); } }; // Sử dụng SegTree seg(n); vector<long long> dp(n, 0); for (int i = 0; i < n; i++) { dp[i] = seg.qry(L[i], R[i]) + a[i]; // query seg.upd(i, dp[i]); // insert }
03 · HLD + Tree DP / DSU on Tree
Kỹ thuật nâng cao

DSU on Tree (Small to Large) & HLD DP

DSU on Tree (Sack): Mỗi nút chỉ "thừa kế" data structure từ con nặng nhất, merge các con nhẹ vào. O(n log n) tổng thể.

HLD + DP: Sử dụng Heavy-Light Decomposition để xử lý query/update trên đường đi cây, kết hợp Segment Tree để tính DP theo chuỗi nặng.

Ứng dụng điển hình:
  • Đếm số màu khác nhau trong cây con — DSU on Tree
  • LCA + DP đường đi trên cây

02 · Kỹ năng Debug DP
⚠️ Lỗi phổ biến

Những lỗi hay gặp nhất

  • Off-by-one: Index từ 0 hay 1? dp[0] có nghĩa gì? Base case sai.
  • Thứ tự tính: dp[i] cần dp[j] đã tính chưa? Vòng lặp đúng chiều chưa?
  • Overflow: dp[i][j] có thể âm vô cùng khi + INF/2?
  • Nhầm min/max: Cần tối đa hay tối thiểu? Khởi tạo ngược.
  • Missing transitions: Quên xét một trường hợp chuyển.
  • Wrong state definition: Trạng thái không đủ thông tin.
💡 Cách debug hiệu quả

Quy trình debug DP

  • In bảng dp ra với test nhỏ (n≤5), kiểm tra thủ công.
  • Viết brute force → so sánh output với DP trên random test.
  • Kiểm tra base case trước, rồi mới kiểm tra transition.
  • Dùng assert để kiểm tra điều kiện bất biến.
  • Thử test corner case: n=0, n=1, tất cả giống nhau, sorted/reverse sorted.

// Template stress test — tìm test case DP sai #include <bits/stdc++.h> using namespace std; mt19937 rng(chrono::steady_clock::now().time_since_epoch().count()); int brute(vector<int>& a) { // Giải O(2^n) hoặc O(n!) brute force int n = a.size(), ans = INT_MIN; for (int mask = 0; mask < (1<<n); mask++) { // ... kiểm tra mọi tập con } return ans; } int solve_dp(vector<int>& a) { // DP cần kiểm tra return 0; } int main() { for (int iter = 0; iter < 100000; iter++) { int n = rng() % 8 + 1; // test nhỏ vector<int> a(n); for (int& x : a) x = rng() % 20 - 10; int expected = brute(a); int got = solve_dp(a); if (expected != got) { cout << "WRONG! n=" << n << "\n"; for (int x : a) cout << x << " "; cout << "\nExpected: " << expected << "\nGot: " << got << "\n"; return 0; } } cout << "All tests passed!\n"; }
03 · Kinh nghiệm phòng thi
1
Đọc đề kỹ — 5 phút đầu
Đọc tất cả bài trước khi làm. Xác định bài nào dễ nhất, làm trước. Gạch chân constraint quan trọng: n≤?, thời gian?, bộ nhớ?
2
Nghĩ trạng thái TRÊN GIẤY
Đừng code ngay. Viết ra: "dp[i] = ...", "chuyển: dp[i] = f(dp[j])...". Verify với ví dụ nhỏ. Tiết kiệm hàng giờ debug.
3
Ước tính độ phức tạp trước
n≤10³: O(n³) ok. n≤10⁴: O(n²) ok. n≤10⁵: O(n log n) ok. n≤10⁶: O(n) ok. Nếu DP O(n²) với n=10⁵ → cần tối ưu (CHT, D&C, deque).
4
Code từng bước nhỏ, kiểm tra ngay
Viết base case → chạy. Thêm transition → chạy. Đừng code 100 dòng rồi mới test. Lỗi nhỏ tích lũy thành thảm họa.
5
Partial score strategy
Nếu không nghĩ ra full solution, code brute force O(2^n) để lấy điểm nhỏ. Với n≤20 thường có subtask bitmask. Điểm nhỏ > 0 điểm.
6
Không bỏ cuộc sớm
Bài DP khó đôi khi cần 30-45 phút để tìm trạng thái đúng. Nếu bí, thử nhìn từ góc độ khác: đổi chiều duyệt, thêm chiều trạng thái, đơn giản hóa bài trước.
